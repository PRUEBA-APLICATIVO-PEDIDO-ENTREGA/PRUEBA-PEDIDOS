🔥 Configurar Firebase

Base de datos en tiempo real + sistema de login

▼
1

Crear proyecto en Firebase

Ve a console.firebase.google.com → haz clic en "Crear un proyecto" → Ponle nombre: don-emilio-pedidos → Desactiva Google Analytics (no lo necesitas) → clic en "Crear proyecto".

2

Agregar una aplicación Web

En la pantalla principal de tu proyecto → clic en el ícono </> (Web) → Nombre: "Don Emilio Web" → NO actives Firebase Hosting → clic en "Registrar app".

Firebase te mostrará un bloque de configuración así. Copia estos valores, los necesitas para el index.html:

// Tu configuración (los valores serán diferentes)
const firebaseConfig = {
  apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXX",
  authDomain: "don-emilio-pedidos.firebaseapp.com",
  projectId: "don-emilio-pedidos",
  storageBucket: "don-emilio-pedidos.appspot.com",
  messagingSenderId: "123456789012",
  appId: "1:123456789:web:abcdef123456"
};
3

Activar Firestore (base de datos)

Menú izquierdo → Build → Firestore Database → "Crear base de datos" → Selecciona "Comenzar en modo de producción" → Ubicación: us-central1 → "Habilitar".

⚠️ En la pestaña "Reglas" de Firestore, pega el contenido del archivo firestore.rules que viene en el paquete. Cambia los emails por los tuyos antes de publicar.
4

Activar Authentication (login)

Menú izquierdo → Build → Authentication → "Comenzar" → Pestaña "Sign-in method" → Habilita "Correo electrónico/contraseña" → Guardar.

5

Crear usuarios (tú y tus clientes)

En Authentication → pestaña "Users" → "Agregar usuario" → crea primero TU cuenta (proveedor) con tu correo y una contraseña segura. Después crea una cuenta por cada tienda cliente.

Ejemplo de usuarios a crear:
• tuCorreo@gmail.com / tuContraseña123 → Proveedor
• tienda.esperanza@gmail.com / pass123 → Tienda La Esperanza
• abarrotes.sol@gmail.com / pass456 → Abarrotes El Sol
6

Pegar tu configuración en index.html

Abre el archivo index.html → busca la sección que dice TU_API_KEY (línea ~20) → reemplaza los valores con los que copiaste en el paso 2.

✅ Después de este paso Firebase ya funciona: guarda pedidos, autentica usuarios y los actualiza en tiempo real.

📊 Google Sheets + Apps Script

Registro automático + correos electrónicos

▼
1

Crear el Google Sheets

Ve a sheets.google.com → crea una hoja nueva → nómbrala "Don Emilio — Pedidos" → en la primera pestaña (abajo) renómbrala a Pedidos (clic derecho → Cambiar nombre).

2

Crear los encabezados manualmente (opcional)

En la fila 1 escribe en cada columna: Referencia | Fecha | Cliente | Email | Productos | Total | Notas | Status | Hora. El Apps Script también los crea automáticamente si no los pones.

3

Abrir Apps Script

En tu Sheets → menú Extensiones → Apps Script → se abre el editor → borra todo lo que hay → pega el contenido completo del archivo apps_script.gs.

4

Configurar tus datos en el script

Al inicio del script cambia estos valores:

const CONFIG = {
  EMAIL_PROVEEDOR: 'tuCorreo@gmail.com',  // ← Tu correo real
  NOMBRE_EMPRESA:  'Don Emilio',           // ← Nombre de tu negocio
  WA_PROVEEDOR:    '523312345678',         // ← Tu WhatsApp (52 + número)
  HOJA_PEDIDOS:    'Pedidos',              // ← No cambiar
};
5

Desplegar como Web App (esto es la clave)

En Apps Script → clic en "Implementar" (botón azul, arriba derecha) → "Nueva implementación" → Tipo: Aplicación web → Ejecutar como: Yo → Quién puede acceder: Cualquier usuario → "Implementar" → Autoriza con tu cuenta de Google.

🔑 Copia la URL de la implementación que aparece. Es algo como: https://script.google.com/macros/s/AKfycb.../exec

Esta URL va a ir en el index.html donde dice TU_APPS_SCRIPT_URL. Con esto cada pedido llega automáticamente al Sheets Y se envían los correos.

6

Agregar la URL al index.html

En el index.html, busca la función enviarPedido(). Después del bloque de Firebase, agrega esta llamada:

// Agregar justo después de: await _fb.addDoc(...)

// Enviar a Google Sheets + correos automáticos
const APPS_SCRIPT_URL = 'https://script.google.com/macros/s/TU_URL_AQUI/exec';
fetch(APPS_SCRIPT_URL, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(pedido),
  mode: 'no-cors'
}).catch(console.error);
7

Probar el sistema

En Apps Script → menú desplegable selecciona probarSistema → clic en ▶️ Ejecutar → acepta permisos → revisa tu Sheets (debe aparecer una fila de prueba) y tu correo (deben llegar 2 emails).

✅ Si ves la fila en Sheets y recibes los correos, ¡el sistema funciona perfectamente!

🐙 GitHub Pages — Publicar el sitio

Hosting gratis con URL profesional

▼
1

Crear repositorio en GitHub

Ve a github.com → clic en "New repository" → Nombre: don-emilio → marca Public → marca "Add a README file" → "Create repository".

2

Subir el index.html

En tu repositorio → clic en "Add file" → "Upload files" → arrastra el archivo index.html → en el campo de commit escribe "Primer versión de la app" → clic en "Commit changes".

⚠️ Asegúrate de haber pegado tu configuración de Firebase ANTES de subir el archivo. El sitio no funcionará si el index.html aún tiene "TU_API_KEY".
3

Activar GitHub Pages

En tu repositorio → pestaña Settings → menú izquierdo: Pages → en "Branch" selecciona main → carpeta: / (root) → clic en "Save".

Espera 2-3 minutos. GitHub Pages te dará una URL como:

https://TU-USUARIO.github.io/don-emilio/
4

Agregar dominio autorizado en Firebase

En Firebase Console → Authentication → pestaña "Settings" → sección "Authorized domains" → "Add domain" → agrega exactamente: TU-USUARIO.github.io

🔴 OBLIGATORIO — Si no haces esto, el login no funcionará en tu sitio publicado.
5

Actualizar el sitio cuando hagas cambios

Edita el index.html en tu computadora → ve a GitHub → tu repositorio → clic en index.html → ícono de lápiz ✏️ → pega el nuevo código → "Commit changes". El sitio se actualiza en ~2 minutos.

💬 WhatsApp — Notificaciones

Alertas instantáneas de nuevos pedidos

▼
1

¿Cómo funciona?

Cuando el cliente envía un pedido, la app abre automáticamente WhatsApp con un mensaje pre-llenado para TI (el proveedor) con todos los detalles del pedido. Sin APIs de pago, sin configuración extra. Funciona con el WhatsApp que ya tienes.

2

Formato del mensaje automático

🛒 *NUEVO PEDIDO — Don Emilio*
Ref: #DE-241203
Cliente: Tienda La Esperanza
Correo: esperanza@gmail.com

📦 Productos:
• Chorizo Rojo (3 kg) — $267
• Queso Oaxaca (2 kg) — $220

💰 Total: $487
📝 Notas: Por favor entregar antes de las 10am
3

Cambiar tu número de WhatsApp

En el index.html, busca la línea:

const WA_PROVEEDOR = '523327963109'; // ← Cambia por tu número
// Formato: 52 + código de área + número (sin +, sin espacios, sin guiones)
// Ejemplo Guadalajara: 52 + 33 + 12345678 = 523312345678

✅ Prueba Final — Checklist

Verifica que todo funciona antes de darlo a los clientes

▼
Orden de verificación recomendada: Firebase → Sheets → GitHub → prueba completa con un pedido real de prueba.
  • Firebase Console → Authentication → aparece al menos 1 usuario
  • Firebase Console → Firestore → existe la base de datos
  • Apps Script → función probarSistema ejecutada sin errores
  • Google Sheets → pestaña "Pedidos" tiene una fila de datos de prueba
  • Correo del proveedor → llegó email con formato de Don Emilio
  • Correo del cliente → llegó email de confirmación
  • GitHub Pages → sitio disponible en la URL pública
  • Firebase Auth → dominio de GitHub Pages autorizado
  • Login funciona en el sitio publicado como cliente
  • Login funciona como proveedor (tu correo)
  • Agregar producto al carrito y llega al total
  • Enviar pedido → abre WhatsApp con el mensaje correcto
  • El pedido aparece en Firestore (Firebase Console → Firestore)
  • El pedido aparece en Google Sheets
  • Como proveedor, el pedido aparece en el panel
  • Como proveedor, cambiar status de "Pendiente" a "Confirmado"
  • Como cliente, ver el cambio de status en "Mis Pedidos" (tiempo real)
🎉 Si todos los puntos están marcados, el sistema está 100% listo para producción.
Ya puedes compartir la URL de GitHub Pages a tus clientes con sus usuarios y contraseñas.

👥 Agregar Nuevos Clientes

Cómo dar acceso a una nueva tienda

▼
1

Crear su cuenta en Firebase

Firebase Console → Authentication → Users → "Agregar usuario" → Correo y contraseña de la tienda.

2

Darles acceso

Comparte con ellos:
• URL del sitio: https://TU-USUARIO.github.io/don-emilio/
• Correo: el que registraste en Firebase
• Contraseña: la que pusiste

3

Cambiar el nombre visible del cliente

Para que en vez de "correo@gmail.com" aparezca "Tienda La Esperanza", en Firebase Auth → Users → clic en el usuario → edita el campo "Display name".

💡 Tip: Puedes dar la misma contraseña inicial a todos y pedirles que no la compartan. Firebase no tiene (en el plan gratis) funcionalidad de "cambiar contraseña" desde el sitio, pero puedes agregar esa función después si la necesitas.

🔧 Problemas Comunes y Soluciones

Si algo no funciona, revisa aquí primero

▼
ProblemaCausaSolución
Login no funciona en GitHub Pages Dominio no autorizado en Firebase Firebase → Auth → Settings → Authorized domains → agregar tu-usuario.github.io
No llegan correos Apps Script no tiene permisos Ejecuta probarSistema → acepta todos los permisos de Google que pida
Los pedidos no se guardan en Sheets URL del Apps Script incorrecta o modo no-cors Verifica que pegaste la URL correcta en index.html. Re-despliega el Apps Script si cambias el código.
Error "permission-denied" en Firestore Reglas de seguridad muy restrictivas Firebase → Firestore → Rules → verifica que los emails del proveedor están correctos
Proveedor no ve los pedidos de otros Su email no está en PROVEEDOR_EMAILS En index.html busca PROVEEDOR_EMAILS y agrega su correo exacto
WhatsApp abre pero el mensaje está vacío El encodeURIComponent falló Abre la consola del navegador (F12) y busca el error. Generalmente es un carácter especial en el nombre del producto.
El sitio muestra página en blanco Error en el index.html Abre F12 → Console → lee el error. Lo más común es que Firebase config tenga comillas mal copiadas.
